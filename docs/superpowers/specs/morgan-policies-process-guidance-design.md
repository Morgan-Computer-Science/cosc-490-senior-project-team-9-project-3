# Morgan Policies And Student Process Guidance Design

## Goal
Make the advisor stronger on official Morgan student-process questions such as transcripts, registrar workflows, transfer credit, withdrawals, overrides, graduation clearance, accommodations, and academic-support procedures.

## Product Intent
This slice should improve the app's ability to answer:
- what do I do
- where do I go
- who handles this process

with grounded Morgan process paths instead of generic advising responses.

## Scope
This pass focuses on four process families:
1. Records and registration
2. Transfer and academic evaluation
3. Course change and exception processes
4. Student support procedures

## Process Families

### 1. Records And Registration
Improve guidance for:
- transcript requests
- registrar workflows
- enrollment verification
- graduation clearance
- academic record updates

### 2. Transfer And Academic Evaluation
Improve guidance for:
- transfer credit questions
- academic evaluation paths
- prior-course fit and articulation-type questions
- transfer work affecting degree progress

### 3. Course Change And Exception Processes
Improve guidance for:
- withdrawals
- add/drop direction
- override or permission-path questions
- registration barriers that require human follow-up

### 4. Student Support Procedures
Improve guidance for:
- accommodations process paths
- counseling/support referrals
- academic difficulty and student-success referrals
- support situations that overlap academics and wellness

## Data Strategy
Use official Morgan pages and offices when available.
If Morgan does not publish an exact named owner for a process, store the nearest valid official office or contact path instead of guessing.

Likely source families for this slice:
- registrar and records pages
- transfer evaluation paths
- advising and registration support pages
- accessibility and counseling pages
- student-success and retention pages

## Retrieval / Routing Expectations
Questions in this slice should route toward process guidance rather than generic degree planning.
Examples:
- transcript -> registrar / records path
- transfer credit -> transfer evaluation path
- withdrawal / override -> course-change or department process path
- accommodations -> accessibility process path
- academically struggling -> student success / CASA / counseling path depending phrasing

## Answer Quality Bar
The app should:
- give a direct official process path if available
- otherwise give the nearest valid Morgan office/contact path
- avoid vague generic answers when the question is clearly procedural

## Boundaries
This pass should not:
- invent deadlines or approval rules
- claim final authority over registrar or department exceptions
- overstate policy details that Morgan does not publish clearly
- turn into a broad legal/policy parsing project

## Success Criteria
This pass is successful if:
- procedural student questions feel more specific and grounded
- the app points to the right Morgan process owner more often
- registrar, transfer, withdrawal, override, and accommodations questions stop falling back to generic advising language
